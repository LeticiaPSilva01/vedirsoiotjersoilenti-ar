MQTT PROTOCOL

MESSAGE QUEUING TELEMETRY TRANSPORT

01. DEFINIÇÃO

O MQTT é um protocolo de transporte de mensagens leve, baseado no modelo de Publicação/Assinatura (Pub/Sub). Ele foi projetado especificamente para conexões remotas onde a largura de banda é limitada e os dispositivos possuem baixo poder de processamento ou energia limitada.

02. ARQUITETURA

Diferente do modelo tradicional cliente-servidor, o MQTT utiliza um Broker como intermediário. As mensagens são organizadas por Tópicos, permitindo que os dispositivos se comuniquem sem estarem diretamente conectados.

PUBLISHER
O dispositivo que envia os dados para o broker em um tópico específico.
BROKER
O servidor central que gerencia e distribui as mensagens recebidas.
SUBSCRIBER
O dispositivo que assina um tópico para receber atualizações em tempo real.

03. SPEC_SHEET

RECURSO VALOR / TIPO
ALCANCE Depende da camada de transporte (Wi-Fi, Ethernet, 4G/5G)
CONSUMO Ultra-baixo (Projetado para "dormir" entre transmissões)
TAXA DE DADOS Baixa (Otimizado para pacotes binários pequenos)
MODELO Publish / Subscribe (Pub/Sub)

VANTAGENS

  • Overhead mínimo nas mensagens (cabeçalhos curtos).
  • Possui níveis de QoS (Qualidade de Serviço) configuráveis.
  • Excelente para conexões instáveis e alta latência.

DESVANTAGENS

  • Ponto único de falha (se o broker cair, a rede para).
  • Não é adequado para envio de arquivos grandes ou streaming.
  • Necessita de um broker ativo o tempo todo na rede.

04. CASOS_DE_USO

Wearables

Monitoramento de saúde enviando pequenos pacotes de dados vitais.

Automação

Controle de lâmpadas, tomadas e sensores térmicos residenciais.

Logística

Rastreamento de frotas e telemetria de veículos em tempo real.
