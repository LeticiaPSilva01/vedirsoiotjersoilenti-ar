HTTP / REST

O protocolo que faz a internet pulsar. No IoT, ele é o mestre da integração, conectando sensores diretamente a APIs e Dashboards.

01. O QUE É O HTTP?

Imagine que o **HTTP** é um mensageiro que leva cartas super detalhadas entre o seu dispositivo e um servidor.

Diferente do LoRaWAN, o HTTP não se importa com a bateria, ele se importa com a informação completa.

NA PROVA:

HTTP significa Hypertext Transfer Protocol. Ele é um protocolo Stateless (sem estado), o que significa que o servidor não guarda memória de requisições anteriores. Cada conversa é uma conversa nova.

  [CLIENTE]             [SERVIDOR]
      |                     |
      |---(Requisição)----->|
      |                     |
      |<----(Resposta)------|
      |                     |
 (A conexão fecha logo após)

02. ANATOMIA DE UMA REQUISIÇÃO

Para ser mestre, você precisa saber o que tem dentro da "carta" que o sensor envia:

HEADERS (O ENVELOPE)

Método: POST

Host: api.meusensor.com

Content-Type: application/json

Auth: Bearer Token123...

BODY / PAYLOAD (O CONTEÚDO)
{
  "sensor_id": "DHT11_Lobby",
  "temperatura": 24.5,
  "status": "online"
}
