HTTP / REST

HYPERTEXT TRANSFER PROTOCOL APPLIED TO IOT

01. DEFINIÇÃO

O HTTP é o protocolo fundamental da Web, mas em IoT ele é amplamente utilizado através de arquiteturas REST (Representational State Transfer). Ele permite que dispositivos enviem e recebam dados utilizando os mesmos métodos que navegadores usam para acessar sites, facilitando a integração com sistemas em nuvem.

02. ARQUITETURA

Diferente do MQTT, o HTTP opera no modelo Cliente-Servidor. O dispositivo IoT (cliente) inicia uma conexão, faz uma requisição ao servidor e aguarda uma resposta.

MÉTODOS (Verbos)
  • GET: Solicita dados do sensor.
  • POST: Envia novos dados/leituras.
  • PUT: Atualiza configurações.
FORMATO

Geralmente utiliza JSON ou XML para estruturar as informações de telemetria entre o dispositivo e a API.

03. SPEC_SHEET

RECURSO VALOR / TIPO
ALCANCE Global (Via Internet TCP/IP)
CONSUMO Alto (Devido ao overhead dos cabeçalhos e conexões persistentes)
TAXA DE DADOS Alta (Capaz de transferir grandes volumes de informação)
ESTADO Stateless (Cada requisição é independente)

VANTAGENS

  • Facilidade de integração com quase qualquer serviço Cloud/Web.
  • Grande suporte de bibliotecas em todas as linguagens.
  • Atravessa firewalls corporativos com facilidade (Porta 80/443).

DESVANTAGENS

  • Alta carga de dados (overhead) para mensagens pequenas.
  • Dificuldade para o servidor "avisar" o dispositivo (falta de push nativo).
  • Consumo de bateria elevado para dispositivos móveis.

04. CASOS_DE_USO

Dashboards Web

Visualização de dados históricos em painéis de controle centralizados.

Câmeras IP

Transmissão de grandes pacotes de dados e streams de mídia.

Gateways Industriais

Sincronização de grandes bancos de dados locais com a nuvem.
